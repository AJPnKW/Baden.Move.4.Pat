# Phase Plan

## Phase 0: Planning To Plan

- define goals, constraints, and deadlines
- identify unknowns that needed research or validation
- determine which choices had decision deadlines

This phase mattered because the project started with ambiguity. The right question was not "what is the final detailed plan?" but "what do we need to know before a detailed plan becomes trustworthy?"

Key outputs:

- initial scope framing
- assumptions and unknowns list
- decision calendar
- first pass at critical dates and dependencies

## Phase 1: Sell-Side Preparation

- prepare the house for photos
- prepare the house for viewings
- sort items into move, sell, donate, discard, and store paths

This phase created a practical separation between "things needed to support the sale" and "things needed for the eventual move." Without that distinction, effort would have become noisy and contradictory.

Key outputs:

- sale-readiness checklist
- staging activity list
- item disposition categories
- issue tracking for prep gaps

## Phase 2: New Home Readiness And Renovation Planning

- assess pre-move work
- compare renovation options, budgets, and timing
- identify what had to happen before and after move-in

This phase was part analysis and part deadline management. It was not enough to ask what changes were desirable; the project had to determine what changes were realistically achievable between possession and actual occupancy needs.

Key outputs:

- renovation option comparison
- room readiness priorities
- before-move vs after-move boundary
- decision log for fixtures, finishes, and sequencing

## Phase 3: Overlap Occupancy Coordination

- plan the overlap period
- separate mover work from self-move work
- sequence staged movement of boxes, furniture, and supplies

The overlap window created both opportunity and complexity. It allowed staged movement, but only if work was intentionally split by method, timing, and dependency.

Key outputs:

- overlap usage plan
- mover vs self-move matrix
- garage sale / donation / discard stream handling
- staged movement checkpoints

## Phase 4: Possession And Immediate Stabilization

- execute day -1, day 0, day +1, and day +2 tasks
- manage utility and connectivity cutovers
- stabilize essential spaces and services

This was the highest-control phase. It functioned like a small cutover window: not everything had to be perfect, but the wrong misses here would have created outsized disruption.

Key outputs:

- possession runbook
- critical-day checklist
- service cutover tracker
- immediate issue-response log

## Phase 5: Post-Move Completion

- resolve deferred tasks
- continue staged unpacking and follow-up work
- close out remaining issues and lessons learned

This phase turned a successful landing into a sustainable transition. It also helped separate real unresolved work from temporary move-day noise.

Key outputs:

- deferred-work list
- remaining punch list
- closeout notes
- lessons learned
