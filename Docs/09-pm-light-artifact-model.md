# PM-Light Artifact Model

## Why This Artifact Set Exists

This project was not managed with a heavy formal methodology, but it still benefited from a compact set of artifacts that kept the work understandable and adjustable.

The point of these artifacts was not paperwork. The point was control.

## Core Artifact Set

### 1. Project Skinny

Purpose:

- define the project in plain language
- align on what success means right now
- stop the project from becoming whatever the latest urgent task happens to be

### 2. Context And Constraints

Purpose:

- identify fixed dates, limitations, and assumptions
- make hidden pressure visible
- separate preference from real constraint

### 3. Phase Plan

Purpose:

- group work into meaningful execution periods
- show where exploratory work ends and tighter sequencing begins
- reduce the feeling that everything is happening at once

### 4. Decision Log

Purpose:

- record important choices
- capture why a choice was made at that time
- reduce revisionist confusion later

### 5. Issue Log

Purpose:

- show what created friction or risk
- distinguish local issues from system-wide issues
- document the response pattern rather than only the problem

### 6. Change Log

Purpose:

- show how the project evolved
- make scope movement visible
- keep later reporting honest

### 7. QA And Validation Summary

Purpose:

- show what was tested or validated
- connect proof-of-concept work to real planning decisions
- make learning visible as part of delivery

### 8. Coordination Thread

Purpose:

- provide a living, lightweight operational thread
- connect status, issues, decisions, and next actions
- create a project-memory layer without heavy admin overhead

## Control Principle

This artifact set follows one main rule:

Use the smallest artifact that creates enough clarity to make the next decision well.

That keeps the method disciplined without making it burdensome.
